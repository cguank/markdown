# LC和take photo的区别
- 背景和问题
大佬，我们需求背景是：扫描二维码的时候，将用户扫描二维码的场景上传给服务器进行反欺诈检测，例如扫描的环境是网站或实体店扫描。
现在遇到的问题是：使用native的相机组件，扫描二维码只会返回二维码code，不会返回拍摄的二维码图片。然后我们在扫描到二维码之后，使用相机进行拍摄的话会有声音，无法做到用户无感知。
我们希望能够在扫描到二维码的时候进行拍摄，并做到用户无感知，请问一下，你们是不是有类似的需求和解决方案？
hi master, we have a requirement, need to scan qr and the photo to analysis AF. and we switch camera mode from QR to TAKE_PHONE. taking photo after scan qr return result, but the device will have a sound and user will perceive it.
#1 do u know if there is a method in RN: scan qr , get the photo without user perception
#2 why do users do LC, the devices dont have a sound ?
- 方案
Hi @guankun.chen | 陈冠坤 |  FE , to answer your questions:
#1 yes, there is a way to return both the QR code and scanned image without user perception, but the native code needs to be changed to support it. There's no workaround with current native implementation
#2 When users do LC, the capture is configured as video recording and each video frame is returned; when users use take photo, the capture is configured as still image capturing, therefore it has the sound.

The way native code can change to support returning both QR code and image, is similar to how LC works, that is video recording at the core. Actually, this approach has been used for ID only for a while, for ID an image can be returned to RN, but there's no request for it before so native didn't return the image. For other countries, when scanning QR code, the capture is configured as metadata capture, so there's no image at all
